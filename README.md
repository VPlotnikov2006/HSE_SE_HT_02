## Домашнее задание №2. Паттерны проектирования.

Исполнитель: Плотников Владимир Денисович

Группа: БПИ-246-2

---

Общая функциональность решения:
- Пользователь имеет возможность добавить/удалить/редактировать категорию/операцию/аккаунт
- Пользователь имеет возможность посмотреть данные в консоли/вывести данные в файл в формате JSON (десериализация не предусмотрена)
- Программа измеряет время выполнения каждой операции и записывает в файл `.log`

---

Возможности для расширения:
- Легко добавить новые способы сериализации данных
- Есть возможность добавления уведомлений 
- Есть возможность добавления новой системы хранения данных (например база данных)

---

Принципы `SOLID` / `GRASP` - старался соблюдать везде, вроде получилось

---

Реализованные паттерны:
- Порождающие
  - Фабричный метод: `BankAccountFactory` и `CategoryFactory` - отвечают за регистрацию в репозиториях и создают `Guid`-ы
  - Строитель: `OperationFactory` - как и фабрики, только из-за большого числа параметров использовал Строителя
  - Одиночка: `INotifier` - для увеличения числа паттернов, можно конечно `ServiceCollection.AddSingleton`
- Структурные
  - Компоновщик: `Group<T>` - для хранения вложенных групп действий пользователя
  - Декоратор: `ActionDecorators` - как в рекомендациях к решению - для обертки пользовательских сценариев
  - Фасад: `IOperation/Category/BankAccountRepository` - для упрощения использования
- Поведенческие
  - Команда: `UserAction`-ы - для отделения пользовательских сценариев от `Application` 
  - Стратегия: `Serializer` - определяет стратегию сериализации данных (Json, в консоль)
  - Шаблонный метод: также `Serializer` - имеет шаблонные методы `Header`, `Footer`, `Separator` и `SerializeObject` и одинаковый для всех метод `Serialize(IEnumerable)`
  - Посетитель: `ISerializationVisitor` - для унифицированной сериализации `Operation`, `Category` и `Bank account`

Паттерн Наблюдатель может быть легко доделан, используя `INotifier` и уведомляя пользователей о подозрительных транзакциях
